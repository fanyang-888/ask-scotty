# Team Conventions

The point of every rule here: a new teammate can clone, run, and contribute
without asking anyone — and every line of code has a traceable origin.

## Branching & commits

- `main` is always runnable (`make test` passes). Work on short-lived branches
  named `<initials>/<topic>` (e.g., `fy/bedrock-provider`), merge via PR.
- Commit messages: imperative summary line ≤ 72 chars. Body explains *why*
  when it isn't obvious.
- **Every commit message carries a provenance tag** (see next section).

## Code provenance (AI use)

Per the course AI Use Modes guide, we label who produced what and what review
it had. The judgment stays with us.

**Commit tags** — first token of every commit subject:

| Tag | Meaning |
|---|---|
| `[human]` | written by a team member without AI |
| `[ai-gen]` | generated by an agent (Operator/Agent mode), human-reviewed before merge |
| `[ai-assisted]` | human-written, AI used as Editor/Refiner or Explainer |

**Review rule:** nothing tagged `[ai-gen]` merges without a **Critic/Red
Teamer** pass by a teammate who did not prompt it — challenge assumptions,
probe edge cases, run the tests locally. Record the review in
[docs/PROVENANCE.md](docs/PROVENANCE.md), which follows the guide's Minimal
Submission Format (AI Use Mode / Prompt(s) / How I improved or changed the
output / What I verified / What I contributed / Final answer submitted).

Code we can't explain is code we don't control — if you can't walk the team
through an AI-generated diff, it doesn't merge.

## Secrets & config

- **Never commit secrets.** Learner Lab credentials rotate every session and
  live only in your shell. `.env*` is gitignored; secrets are injected at
  runtime, not build time.
- Configuration is not secrets: model selection (`MODEL_PROVIDER`,
  `BEDROCK_MODEL_ID`), table names, and endpoints are environment variables
  set by the SAM template.

## Cloud resources

- Everything is created by `infra/template.yaml` — console clicking is for
  exploration only ("ClickOps for learning, code for operating").
- Every resource carries the four cost-allocation tags: `Owner`, `Project`,
  `Environment`, `CostCenter`. Untagged resources are ungoverned resources.
- Region: `us-east-1` unless the team decides otherwise (Bedrock
  availability + Learner Lab default).
- Learner Lab has a hard budget ceiling — prefer `make invoke`/`make test`
  (free, local) over deployed testing; delete stacks you're done with
  (`sam delete`).

## Style

- Python 3.12, standard library first; add a dependency only when you feel
  the pain. Tests live next to the code they test (`api/tests/`).
- The system prompt is a spec: it is versioned
  (`api/src/ask_scotty/prompts/system_prompt.md`), covered by tests, and
  bumped like code.
